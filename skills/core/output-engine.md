# 🧾 输出解释层

本文件定义了如何将原始 JSON 响应转换为用户可读的信息。
所有 AI 生成的输出必须遵循本规范。

---

## 1. 第一性原则

**输出不是汇报数据，是帮用户做判断。**

原始数据 → 你的分析 → 输出判断

每条输出应该遵循：
```
关键结论（如果有清晰发现）
└─ 核心数据（表格/列表，突出角色关注的字段）
   └─ 异常提醒（如果有）
      └─ 建议动作（如果有）
```

---

## 2. 格式化规则

### 数据展示

| 数据类型 | 格式 | 示例 |
|---------|------|------|
| 列表 | 表格，突出角色关注字段 | `│ 名称 │ 负责人 │ 金额 │ 状态 │` |
| 单条详情 | 关键字段摘要 + 其他字段折叠 | |
| 统计数据 | 总览 + 明细 | `本月合同 13 份，总金额 ¥285 万` |
| 错误/空 | 说明原因，不贴原始 JSON | `暂无数据（当前过滤条件下）` |

### 表格规则
- 使用标准 Markdown 表格（不要手工画 ASCII 框线）
- 列数 ≤ 5，多字段时分批展示
- 关键值（金额、日期、状态）放在前列

### Emoji 状态指示
| emoji | 含义 |
|-------|------|
| 🟢 | 正常、已完成、活跃 |
| ⚠️ | 需要注意、即将过期 |
| 🚨 | 紧急、逾期、异常 |
| ✅ | 已完成、已处理 |
| 📋 | 计划、提醒 |
| 📊 | 统计、概览 |

---

## 3. 角色感知的输出适配

根据角色调整展示优先级：

### 销售
- 优先展示：负责人、跟进时间、阶段、下一步动作
- 输出侧重：操作建议，具体到"联系谁 + 做什么 + 先后顺序"
- 摘要型列表优先，支持点击查看详情

### 经理
- 优先展示：团队统计、排名、覆盖率、目标进度
- 输出侧重：管理决策，结构化风险 + 建议
- 从宏观到微观：全貌 > 个人 > 具体记录

### 财务
- 优先展示：金额汇总、日期、发票状态、逾期天数
- 输出侧重：数值精确，金额必须带单位
- 异常项优先排列，正常项折叠

---

## 4. 输出反模式（禁止）

```
❌ 直接贴 JSON 响应
   {"code":100200,"data":{"list":[...],"total":13}}

❌ 纯搬运不做判断
   "您有 13 条线索，创建时间是···"

❌ 抛给用户选择但不给建议
   "有 3 条线索超期未跟进，您想先看哪个？"

❌ 表格列太多导致换行（超过 5 列分两批展示）
   "│ ID │ 名称 │ 负责人 │ 金额 │ 阶段 │ 创建时间 │ 跟进时间 │···"
```

## 5. 输出示例

### 好（销售视角）
```
📋 您本月有 13 条线索，其中 3 条超过 48 小时未跟进：

| 客户名称 | 创建时间 | 状态 |
| --- | --- | --- |
| XXX 科技 | 05-06 | 🟢 新 |
| YYY 集团 | 05-02 | ⚠️ 超期 |

⚠️ YYY 集团已 7 天未跟进，建议今日优先联系
```

### 好（经理视角）
```
📊 部门本月线索概览

| 成员 | 新增 | 跟进率 | 转化 |
| --- | --- | --- | --- |
| 张三 | 8 | 75% | 12% |
| 李四 | 5 | 40% | 8% ⚠️ |

🚨 李四跟进率低于团队均值（68%），建议 1v1 沟通
```

### 好（财务视角）
```
💰 本月合同统计

| 状态 | 数量 | 金额 |
| --- | --- | --- |
| 已签约 | 5 | ¥86 万 |
| 待回款 | 3 | ¥42 万 |
| 逾期 | 1 | ¥15 万 🚨 |

🚨 合同 "XX 项目" 回款已逾期 15 天，金额 ¥15 万
```

## 6. 多模块搜索输出格式（全局模糊搜索）

当执行跨模块全局搜索（cli-spec.md §12）时，输出须遵循以下格式。

### 6.1 输出结构

```text
🔍 全局搜索："{搜索词}"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📌 {模块中文名}（{命中条数} 条）
{数据表格或说明}

📌 {模块中文名}（{命中条数} 条）
{数据表格或说明}

...（所有有数据的模块按优先级排列；无数据的在最下方折叠）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 汇总：共找到 {总数} 条匹配记录
{根据角色给出的行动建议（如有）}
```

### 6.2 输出规则

| 规则 | 说明 |
|------|------|
| **有数据才展示** | 命中条数 = 0 的模块不展开表格，仅以单行显示无匹配 |
| **按优先级排序** | 线索、线索池、客户在前（高优），商机、公海居中，联系人在后 |
| **模块用 emoji 标记** | 📌 统一前缀，**不加模块名重复 emoji** |
| **表格 ≤5 列** | 按模块特性选择关键字段展示 |
| **汇总行** | 包含总命中数 |
| **行动建议** | 如果某个模块命中多且有明显趋势，给出操作建议 |
| **失败/超时** | 标注为 `⚠️ {模块} 查询失败` 或 `⚠️ {模块} 查询超时` |

### 6.3 各模块表格列定义

| 模块 | 关键列（≤5 列） |
|------|----------------|
| 线索 | 名称 / 公司 / 电话 / 负责人 / 创建时间 |
| 线索池 | 名称 / 公司 / 电话 / 负责人 / 创建时间 |
| 客户 | 名称 / 行业 / 省份 / 负责人 / 创建时间 |
| 商机 | 名称 / 金额 / 阶段 / 负责人 / 创建时间 |
| 公海 | 名称 / 行业 / 省份 / 负责人 / 创建时间 |
| 联系人 | 姓名 / 公司 / 手机 / 邮箱 |

### 6.4 空结果处理

```text
📌 公海（0 条）
（无匹配结果）
```

### 6.5 失败/超时处理

```text
📌 线索池 ⚠️ 查询超时
📌 公海 ⚠️ 查询失败
```

📌 线索（8 条）
| 名称 | 公司 | 电话 | 负责人 | 创建时间 |
| --- | --- | --- | --- | --- |

### 6.7 行动建议规则

| 场景 | 建议内容 |
|------|---------|
| 线索 + 客户同时命中 | `💡 客户"{名称}"名下有关联线索和商机，建议进一步查看客户详情` |
| 线索池命中多 | `💡 线索池中有 {N} 条可领取线索，建议先看看` |
| 联系人命中多 | `💡 建议按公司/部门筛选联系人以缩小范围` |
| 仅 1 个模块有数据 | `💡 仅在 {模块} 中匹配到 {N} 条结果，是否需要扩大搜索范围？` |
| 无任何匹配 | `💡 未找到与"{搜索词}"相关的任何记录，请检查关键词是否正确` |
| 超过 30 条总结果 | `💡 搜索结果较多（{N} 条），建议增加筛选条件或指定模块` |
