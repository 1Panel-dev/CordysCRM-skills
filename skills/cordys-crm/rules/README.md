# 📋 自定义规则目录

本目录用于存放 Cordys CRM 模块的自定义规则，AI 在对应场景下自动加载。

---

## 目录结构

```
rules/
├── README.md                  # 本文件
├── form-rules/                # 表单校验规则
│   ├── account.md             # 客户表单自定义校验
│   ├── lead.md                # 线索表单自定义校验
│   ├── opportunity.md         # 商机表单自定义校验
│   └── contact.md             # 联系人表单自定义校验
├── field-mapping/             # 字段映射规则
│   ├── lead-to-account.md     # 线索→客户字段映射
│   └── lead-to-opportunity.md # 线索→商机字段映射
└── business-rules/            # 业务规则
    ├── opportunity.md         # 商机业务约束
    └── contract.md            # 合同业务约束
```

## 加载机制

AI 在执行对应操作时，按以下优先级加载规则：

```
1. API 返回的表单定义（系统规则，始终加载）
2. rules/form-rules/{module}.md（自定义校验规则，存在则加载）
3. rules/field-mapping/{场景}.md（自定义字段映射，存在则加载）
4. rules/business-rules/{module}.md（自定义业务规则，存在则加载）
```

- 自定义规则**增强但不覆盖**系统规则
- 文件不存在时静默跳过，不报错
- 文件内容为空或格式无效时，等同于不存在

## 文件格式规范

### form-rules/{module}.md

定义表单字段的自定义校验规则：

```markdown
# {模块} 表单校验规则

## 必填字段补充
API 表单定义中「建议填写但非必填」的字段，在此设为实际必填：
- `phone`：实际业务中电话必填
- `industry`：行业必填，不允许为空

## 值约束
定义字段的值范围（比 API 表单更严格）：
- `amount`：商机金额 > 0，且 ≤ 10,000,000
- `name`：长度 2-100 字符，不允许纯数字

## 字段联动
定义字段间的依赖关系：
- 当 `stage` = "已签约" 时，`signTime` 必填
- 当 `source` = "转介绍" 时，`referrer` 必填

## 默认值
定义特定场景下的默认值：
- `ownerId`：默认当前用户
- `departmentId`：默认当前用户所属部门
```

### field-mapping/lead-to-{target}.md

定义线索转化时的字段映射关系：

```markdown
# 线索 → {目标模块} 字段映射

## 直接映射
| 线索字段 | 目标字段 | 说明 |
|---------|---------|------|
| company | name | 公司名称直接映射 |
| phone | phone | 电话号码直接映射 |

## 转换映射
对值做转换后再映射：
| 线索字段 | 目标字段 | 转换规则 |
|---------|---------|---------|
| province | area | 省份 → 大区：广东→华南 |

## 默认值
转化时自动填充的值：
| 目标字段 | 默认值 | 说明 |
|---------|--------|------|
| source | "线索转化" | 标记来源 |
| ownerId | 当前用户 | 默认归属当前用户 |
```

### business-rules/{module}.md

定义业务层面的约束规则：

```markdown
# {模块} 业务规则

## 操作约束
- 商机阶段只能向前推进，不能回退（除非管理员）
- 合同金额修改超过 20% 需经理审批

## 关联约束
- 创建商机时必须关联客户
- 联系人必须关联客户

## 状态机
定义合法状态流转：
- 线索：新建 → 已联系 → 已确认 → 已转化
- 商机：初步接触 → 需求确认 → 方案报价 → 谈判 → 赢单 / 输单
```

## 使用示例

### 示例 1：客户表单必填规则

创建 `rules/form-rules/account.md`：

```markdown
# 客户表单校验规则

## 必填字段补充
- `phone`：电话为实际必填
- `industry`：行业为实际必填

## 默认值
- `ownerId`：默认当前用户
```

AI 创建客户时，即使 API 表单定义中 phone/industry 不是 required=true，也会按此规则强制要求填写。

### 示例 2：商机金额约束

创建 `rules/business-rules/opportunity.md`：

```markdown
# 商机业务规则

## 值约束
- `amount`：0 < amount ≤ 50,000,000（¥5000万上限）

## 阶段流转约束
- "赢单" → 不可回退
- "输单" → 必须填写输单原因
```

AI 创建/更新商机时会自动应用这些约束。
