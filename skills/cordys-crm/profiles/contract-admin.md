# 商务 / 合同管理角色配置

> 匹配规则见 core/role-engine.md
>
> 匹配关键词：商务、合同管理、合同专员、法务、合规、商务经理、商务总监

## 核心关注
- **合同全生命周期**：起草中、审批中、已签约、执行中、即将到期、已归档
- **合同条款**：金额、付款方式、交付条款、违约责任
- **审批进度**：合同审批流当前节点、谁卡住了、驳回原因
- **到期与续约**：合同即将到期、需要续约谈判
- **变更记录**：合同变更历史、补充协议
- **开票对接**：签约后是否已触发开票流程（衔接财务）

> **和财务的区别**：财务看"钱回来了没有"，商务看"合同签对了没有、有没有风险、要不要续约"。

## 默认查询偏好
| 场景 | 推荐命令 |
|------|---------|
| 合同列表（全部） | `crm page contract` |
| 审批中的合同 | `crm page contract '{"combineSearch":{"conditions":[{"value":"待审批","operator":"EQUALS","name":"approvalStatus","type":"SELECT"}]}}'` |
| 本月签约合同 | `crm page contract '{"combineSearch":{"conditions":[{"operator":"DYNAMICS","name":"startTime","value":"MONTH","type":"TIME_RANGE_PICKER"}]}}'` |
| 即将到期合同 | `crm page contract '{"combineSearch":{"conditions":[{"value":[now_ts, now_ts+30*86400000],"operator":"BETWEEN","name":"endTime","type":"DATE_TIME"}]}}'` |
| 合同详情 | `crm get contract <ID>` |
| 合同审批进度 | `crm approval resource detail <resourceId>` |
| 工商抬头 | `crm page contract/business-title` |
| 报价单（合同前链路） | `crm page opportunity/quotation` |

> 审核状态=`approvalStatus`，开始时间=`startTime`，结束时间=`endTime`。

## L2C 典型工作流

### 日常

#### 合同审批追踪（"审批到哪了"）
```
执行：
  1. POST /contract/page + conditions: approvalStatus 为审批中
  2. 对每个合同 GET /approval-resource/detail/{resourceId}
输出：审批中合同 + 当前节点 + 持续时间
```

#### 今日待签（"今天签了什么"）
```
执行：
  POST /contract/page + conditions: startTime=TODAY
输出：今日签约列表 + 金额汇总
```

#### 驳回处理（"被驳回的"）
```
执行：
  1. 查审批列表，筛选驳回状态
  2. 获取每条审批的驳回原因
  3. 按驳回次数排序
输出：驳回合同 + 原因 + 建议
```

### 周常

#### 合同周报（"本周签约汇总"）
```
执行：
  1. 本周签约合同：数量 + 金额汇总
  2. 按部门分类
  3. 按客户分类
输出：签约概览 + 部门分布 + 客户分布
```

#### 到期预警（"哪些快到期了"）
```
执行：
  POST /contract/page + conditions: endTime BETWEEN [now, now+30d]
  按 endTime 升序排列
输出：到期合同列表 + 客户 + 金额 + 续约建议
```

### 月常

#### 月度合同统计（"本月签约月报"）
```
执行：
  1. 本月签约数量 + 金额 + 环比
  2. 合同类型分布
  3. 平均审批周期
输出：签约趋势 + 类型分布 + 审批效率
```

#### 续约扫描（"下月到期合同"）
```
执行：
  1. 未来 30-60 天到期的合同
  2. 按客户汇总
  3. 标记有无续约意向
输出：到期合同 + 客户 + 续约建议
```

#### 合规审计（"合同完整性检查"）
```
检查项：
  1. 签约合同是否有盖章
  2. 签约合同是否有附件
  3. 合同是否有完整的审批记录
  4. 合同金额是否与商机金额一致
输出：合规检查清单 + 问题合同列表
```

## 交互模式
- **默认输出**：合同状态和日期优先，审批流次之，金额辅助
- **数据深度**：合同列表 → 单条详情 → 审批流详情
- **提醒风格**：强调日期（到期、超期审批）和流程完整性
- **行动建议**：审批催办、到期续约提醒、条款风险标注
- **链路视角**：查看合同时自动检查审批进度和到期状态

## KPI 基准线
| 指标 | 正常 | 警戒 | 严重 |
|------|------|------|------|
| 合同审批周期 | ≤ 3 天 | 3-7 天 ⚠️ | > 7 天 🚨 |
| 合同到期提前关注 | ≥ 60 天 | 30-60 天 ⚠️ | < 30 天 🚨 |
| 合同驳回率 | ≤ 10% | 10-20% ⚠️ | > 20% 🚨 |
| 签约后开票触发 | ≤ 3 天 | 3-7 天 ⚠️ | > 7 天 🚨 |
| 合同附件完整率 | 100% | < 100% ⚠️ | — |

## 跨角色协作
| 触发条件 | 动作 |
|---------|------|
| 合同审批通过（签约） | 自动提醒 **销售** "合同已签，可推进交付" + 提醒 **财务** "创建回款计划" |
| 合同被驳回 ≥ 2 次 | 提醒 **销售经理** "合同{名称}审批卡顿，需协调" |
| 合同即将到期（≤ 30天） | 提醒 **销售** "客户{名称}合同将到期，是否续约" |
| 合同金额 > ¥100 万 | 提醒 **高管** "大额合同待审" |

## 权限边界
| 能做 | 不能做 |
|------|--------|
| 查看所有合同 | 修改线索/商机数据 |
| 追踪合同审批流 | 查看非关联模块的跟进记录 |
| 查看工商抬头和发票 | 发起回款操作 |
| 查看报价单（合同上游） | 查看其他部门的销售数据 |

## 角色内子类型
| 子类型 | 关键词 | 差异 |
|--------|--------|------|
| 商务经理 | 商务经理、商务总监 | 关注团队合同量和签约指标 |
| 合同专员 | 合同专员、合同管理 | 关注条款细节和流程完整性 |
| 法务 | 法务、合规 | 关注条款风险而非签约量 |

> 可通过 `ROLE_MAP` 创建独立 profile，如 `商务经理|商务总监=contract-admin-manager`。

## 异常预警
- 🚨 合同审批超过 7 天未完成
- 🚨 合同被同一节点驳回 ≥ 2 次
- ⚠️ 合同到期 ≤ 30 天且未标记续约
- ⚠️ 签约合同缺附件/缺审批记录
- ⚠️ 本月签约量环比下降 > 30%
